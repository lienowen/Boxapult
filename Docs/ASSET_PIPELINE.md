# Parcel Patrol Asset Pipeline

## Principle

Production art must never be drawn directly into gameplay code. Source artwork is exported as separate transparent PNG files, validated, and then packed into a Phaser texture atlas. Runtime names remain stable even when artwork is replaced.

## Coordinate and scale standard

- Logical game resolution: 1280 × 720.
- Production source scale: 2×.
- Runtime atlas scale: 1× unless high-density UI needs 2×.
- Gameplay objects use a centered pivot.
- Ground and environment tiles use bottom-center alignment.
- Transparent padding: 4–8 px at source scale, never large empty canvases.

## Background layers

Each route uses reusable layers rather than one complete screenshot:

- `sky-gradient`: generated by code/configuration.
- `city-far`: seamless 2048 × 512 tile.
- `city-near`: seamless 2048 × 640 tile.
- `route-overlay`: optional silhouettes such as cranes, signs or antennae, 2048 × 720.
- clouds and small atmosphere effects remain reusable particles.

## Naming

Use lowercase kebab-case and semantic animation names:

```text
player-idle-01.png
player-fire-01.png
enemy-scout-fly-01.png
enemy-raider-fire-01.png
fx-explosion-01.png
bg-downtown-near.png
ui-icon-shield.png
```

Do not use `final`, `new`, `copy`, Chinese filenames, dates or version numbers in runtime asset names.

## Repository layout

```text
assets/source/     editable, separate PNG masters and source notes
assets/runtime/    packed atlas PNG/JSON and optimized backgrounds
src/config/assetManifest.js
```

## Atlas rule

- Pack gameplay sprites into `gameplay-atlas.png` and `gameplay-atlas.json`.
- Pack UI into a separate `ui-atlas.png` and `ui-atlas.json`.
- Background tiles remain separate compressed images.
- Gameplay code references frame names from `assetManifest.js`, never raw file paths.

## Acceptance checks

An asset is rejected when:

1. its visible object scale conflicts with the specification;
2. pivot or transparent padding differs from its family;
3. lighting direction conflicts with other gameplay objects;
4. outlines, perspective or rendering style differ;
5. an animation changes object size between frames;
6. the object cannot be recognized at the runtime display size;
7. it includes text that should be localized or rendered by UI code.
